# BEO Operator Card

## Normal path

`beo-explore -> beo-plan -> beo-validate -> beo-execute -> beo-review -> done`

## Support

Debug proves unproven root cause.
Route repairs unsafe owner/feature identity.
Setup handles setup/check/usage only.
Reference returns canonical references only.

## Mutation

Only execute mutates product files.
Execute requires `PASS_EXECUTE`, `approval_ref`, selected execution set, declared scope, verification contract, and verified integrity.

## Review

Review uses cold evidence.
Review does not implement fixes.

## Learning

Compound records one observed case.
Dream consolidates repeated finalized cases.
Author edits/creates skills only when explicitly requested or selected.
Learning never grants approval, mutation, review acceptance, routing, or Human Gate fallback.

## State

STATE/HANDOFF are display/orientation only and lose to current required surfaces.

## Commands

Command names are not doctrine. Workflow-visible command behavior lives in `tool-contracts.md`.

## Go-mode

Go-mode changes assumption posture only. It does not bypass Human Gates, approval, validation, integrity, execution scope, review, or learning boundaries.
