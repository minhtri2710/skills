# Artifact Density

Authority: canonical compact/single-ticket and full density selection. Density changes ceremony only; it never weakens approval, Human Gates, owner boundaries, execution scope, or review.

## Compact / single-ticket

Use compact when work is one bounded item, a small file set, no complex rollback, no broad risk, no complex generated outputs, and one normal execution set.

Compact uses fewer runtime artifact files and one structured `beo.ticket` authority block in `TICKET.md`.

## Full

Use full for multi-item work, repair or rollback execution sets, broad risk, complex generated outputs, complex verification, or any case where separate CONTEXT, PLAN, TRACKER, and REVIEW surfaces reduce ambiguity.

## Selection rule

Choose the smallest density that preserves authority. If compact eligibility is unclear, use full.
