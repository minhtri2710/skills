# BEO Learning v2

## Review default

Accepted closure defaults to `done`.

Route to `beo-compound` only when `REVIEW.md` records `durable-candidate` or `unclear` learning evidence.

Do not route to `beo-compound` merely to decide whether learning exists.

## beo-compound

`beo-compound` records one feature-level learning outcome. It does not promote shared guidance and does not reopen review.

## beo-dream

`beo-dream` requires either:

- at least two accepted/finalized feature outcomes supporting the same shared pattern; or
- explicit user instruction requesting corpus-level consolidation.

One accepted feature is not enough.

Do not mutate shared guidance without current user-confirmed target path.

Learning must never reopen runtime delivery, mutate implementation, or promote shared guidance from one feature alone.
