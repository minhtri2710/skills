# Content Guidelines

## Anti-Slop Rules

Avoid the default AI visual soup:

- generic purple or cold-blue gradients
- faceless vector people
- oversized rounded cards with no concept
- default dashboard composition for non-dashboard tasks
- silhouettes used instead of real brand assets
- decorative motion that says nothing

## Positive Defaults

Prefer:

- a clear concept first
- strong type hierarchy
- fewer stronger colors
- real product imagery when the brand matters
- meaningful contrast
- whitespace with purpose
- motion tied to story, hierarchy, or state change

## Placeholder Rules

Use placeholders only when a missing asset blocks correct work.

If you use one:

- label it internally
- tell the user
- replace it as soon as the real asset arrives

An honest placeholder is better than fake detail.

## Typography

- Choose a display face with intent.
- Use body type that supports reading, not branding noise.
- Do not default to Inter for display unless the product already does.
- Tighten or loosen rhythm based on the composition, not habit.

## Color

- Start from brand colors when available.
- If inventing a palette, commit to a direction.
- Use one dominant accent unless the concept truly needs more.
- Avoid neon-dark tech styling unless the brief explicitly calls for it.

## Layout

- Match layout to medium.
- Decks should feel like slides, not browser apps.
- Infographics should feel editorial or diagrammatic, not like landing pages.
- App prototypes should feel like products, not feature posters.

## Motion

- Animate hierarchy and reveal.
- Use motion to help the eye.
- Do not animate everything equally.
- Use restrained timing and strong easing.

## Variations

Each variation should test a specific hypothesis:

- different type voice
- different density
- different emotional temperature
- different visual logic

Do not create three versions that only change the accent color.
