# Scene Templates

Use these when the user names an output type but does not provide a full composition.

## Cover

Good for:

- article covers
- keynote title slides
- announcement posters

Structure:

1. one anchor idea
2. one dominant type moment
3. one supporting diagram or image
4. one secondary metadata line

## Slide Data Page

Good for:

- benchmark summaries
- launch numbers
- product performance updates

Structure:

1. one hero metric
2. one comparison view
3. one key takeaway
4. one small data source note

## Vertical Infographic

Good for:

- before and after stories
- process diagrams
- data summaries

Structure:

1. a strong top takeaway
2. 2 to 4 sections
3. a paced reading flow
4. repeated visual grammar

## Landing Hero

Good for:

- launch pages
- portfolio intros
- product homepages

Structure:

1. claim
2. proof
3. action
4. one supporting visual system

## App Prototype

Good for:

- onboarding
- core flows
- settings or dashboard concepts

Structure:

1. real device shell
2. 3 to 5 meaningful states
3. a single clear flow
4. tappable transitions

## Developer Docs Page

Good for:

- API landing pages
- product docs
- technical overviews

Structure:

1. navigation or table of contents
2. one hero code example
3. grouped feature blocks
4. low-friction reading hierarchy

## Motion Hero

Good for:

- launch videos
- capability explainers
- concept pieces

Structure:

1. opening premise
2. 2 to 4 narrative beats
3. a closing brand or takeaway state
4. a still final frame that can be held in export
